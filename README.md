# МедАналитика: Отчеты 

Приложение MedAnalyticsReportsApp - тестовый модуль для формирования аналитического отчета работы бригад скорой медицинской помощи (СМП) 


## Описание проекта

Приложение позволяет выгружать структурированную информацию о работе дежурных бригад, фильтровать данные по ролям сотрудников (врач, фельдшер, водитель), временным периодам и ФИО, экспортировать данные в Excel

---

## Технологический стек

* Среда разработки: RAD Studio Delphi 11
* Компоненты UI: DevExpress VCL
* Доступ к данным: FireDAC 
* СУБД: Firebird 5

---

## Архитектура проекта: Состав модулей и слоев

Проект реализован по трехслойной архитектуре (близкой к MVC) с выделением сервисного слоя и хелперов.

### 1. Слой представления (Views)
* **marMainForm.pas (frmMain):** Главная форма приложения. Служит контейнером для фреймов отчетов и управляет общей структурой интерфейса.
* **marBrigadeReport.pas (fraBrigadeReport):** Визуальный фрейм (TFrame) отчета по бригадам СМП. Содержит компоненты TcxGrid и TcxDBTableView. Отвечает за отображение данных, цветовое условное форматирование по ролям, UI-контролы фильтрации, а также кнопки запуска генерации отчета и экспорта в Excel.

### 2. Слой бизнес-логики и контроллеров (Controllers)
* **marMainController.pas:** Главный контроллер приложения. Координирует глобальные процессы и инициализацию.
* **marBrigadeReportController.pas:** Контроллер отчета бригад. Инкапсулирует компоненты TFDQuery и TDataSource, управляя их жизненным циклом и поставляя данные для View. Динамически загружает внешний SQL-шаблон T_BrigadeReport.sql (позволяет менять логику без пересборки). Отвечает за параметризацию запроса: передачу дат, фильтра ФИО и маппинг ролей сотрудников в ID базы данных (4, 5, 6).
* **marBrigadeReportFilter.pas:** Объектная модель фильтрации. Хранит, валидирует и передает параметры отчета (период дат, роли).

### 3. Слой сервисов и доступа к данным (Services)
* **marDBConnectionService.pas:** Сервис управления жизненным циклом и пулом физических подключений к СУБД Firebird 5.
* **marDatabaseDM.pas (dmDB):** Глобальный дата-модуль. Содержит центральные компоненты TFDConnection и TFDPhysFBDriverLink для работы с Firebird 5 через FireDAC. Передает экземпляр соединения в контроллеры (включая TBrigadeReportController) для выполнения локальных запросов, обеспечивая единую точку авторизации.

### 4. Вспомогательный слой (Helpers & Resources)
* **marUtils.pas / marConverters.pas:** Общие утилиты и безопасные конвертеры типов.
* **marExportGridToExcel.pas:** Хелпер для выгрузки данных из TcxGrid в Excel (XLSX) с сохранением форматирования.

### 5. Слой моделей (Models) & Архитектурные допущения
Так как проект является тестовым модулем и не предполагает дальнейшего масштабирования, архитектура намеренно упрощена:
* Классы DTO и отдельный сервис выборки данных из базы не выделялись.
* Компоненты TFDQuery и TDataSource создаются напрямую в контроллере отчета и связываются с колонками грида во View.

## Развертывание приложения

Для корректной работы скомпилированного приложения MedAnalyticsReportsApp.exe в корневом каталоге должны находиться следующие элементы:

* **fbclient:** Папка с клиентскими библиотеками СУБД Firebird 5.
* **Templates:** Папка для хранения внешних SQL-шаблонов. Содержит файл T_BrigadeReport.sql, который является основным шаблоном для построения отчета.
* **export:** Папка, используемая по умолчанию для сохранения отчетов Excel (XLSX). Содержит файлы с экспортом тестовых данных.
* **settings.config:** Основной файл конфигурации приложения.

### Настройка подключения к БД
Перед первым запуском необходимо отредактировать файл settings.config в корневом каталоге, указав актуальные параметры подключения к базе данных.

Пример содержимого файла settings.config:

```ini
[Database]
Server=localhost
Port=3050
Path=W:\FDB\SYSTEM200626.FDB
CharacterSet=UTF8
```
